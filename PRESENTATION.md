## Лойиҳа презентацияси

## 1. ЛОЙИҲА ҲАҚИДА УМУМИЙ МАЪЛУМОТ

### 1.1 Мақсад
LMS — олий таълим муассасаларида ўқув жараёнини рақамлаштириш, талабалар баҳоларини ҳисобга олиш, мустақил ишларни бошқариш ва таълим сифатини мониторинг қилиш учун мўлжалланган замонавий веб-тизим.

### 1.2 Технологиялар стеки
| Қатлам | Технология |
|--------|-----------|
| **Backend** | Laravel 11.9 (PHP 8.2+) |
| **Frontend** | Blade + Alpine.js + Tailwind CSS + Flowbite |
| **Маълумотлар базаси** | MySQL / MariaDB (Eloquent ORM) |
| **Реал-вақт компонентлар** | Livewire 3.5 |
| **Ҳуқуқлар тизими** | Spatie Permission (RBAC) |
| **Маълумот импорт/экспорт** | Maatwebsite Excel, PHPOffice |
| **Интеграциялар** | HEMIS API, Moodle, Telegram Bot |

### 1.3 Фойдаланувчилар
Тизимда **уч хил портал** мавжуд, ҳар бири алоҳида аутентификация (guard) тизимига эга:

- **Админ панели** — суперадмин, маъмурлар, деканлар, кафедра мудирлари, ўқитувчилар ва бошқа ходимлар
- **Ўқитувчи портали** — ўқитувчилар учун дарслар, баҳолар ва топшириқлар бошқаруви
- **Талаба портали** — талабалар учун баҳолар, давомат ва топшириқларни кўриш

## 2. ЖУРНАЛ БЎЛИМИ

### 2.1 Тавсиф
Журнал — тизимнинг марказий бўлими бўлиб, у талабаларнинг **давомати ва баҳоларини кунлик ҳисобда** кўрсатади. Бу бўлим орқали ўқитувчилар ва маъмурият ҳар бир талабанинг семестр давомидаги ўқув кўрсаткичларини реал вақтда кузатиб боради.

### 2.2 Ишлаш принципи

```
Фильтрлар танлаш → Гуруҳ/Фан/Семестр → Батафсил журнал кўриниши
      │                                            │
      ▼                                            ▼
  Каскадли фильтр:                         Журнал жадвали:
  Факультет → Йўналиш →                   ┌──────────────────────┐
  Курс → Семестр →                         │ Талаба | Сана1 | ... │
  Гуруҳ → Фан                             │ Иванов |  85   | ... │
                                           │ Петров |  90   | ... │
                                           └──────────────────────┘
```

**Қадамлар:**

1. **Фильтрлаш** — фойдаланувчи каскадли фильтрлар орқали маълумотни танлайди:
   - Таълим тури → Факультет → Йўналиш → Курс → Семестр → Гуруҳ → Фан
   - Ҳар бир танлов кейинги фильтр учун маълумотларни AJAX орқали динамик юклайди
2. **Кўрсатиш** — танланган гуруҳ ва фан бўйича батафсил журнал очилади
3. **Баҳолар жадвали** — ҳар бир дарс куни бўйича ЖБ (жорий баҳо) ва амалий машғулот баҳолари кўрсатилади
4. **Ўртача баҳо** — кунлик баҳолар асосида ўртача ҳисобланади

### 2.3 Асосий имкониятлар

| Имконият | Тавсиф |
|----------|--------|
| **Каскадли фильтрлар** | 7 босқичли фильтрлаш тизими (факультет → фан) — ҳар бир босқич динамик юкланади |
| **Кунлик баҳолар** | Ҳар бир дарс санаси бўйича ЖБ (жорий баҳо) ва амалий баҳолар |
| **Ўртача ҳисоб** | Кунлик баҳолар асосида автоматик ўртача ҳисоб-китоб |
| **Мустақил таълим баҳоси** | Журнал ичидан МТ баҳосини бевосита қўйиш имконияти |
| **Қайта топшириш** | Қайта топшириш (retake) баҳоларини бошқариш |
| **Файл юклаш** | Талабалар юборган файлларни кўриш ва юклаб олиш |
| **Ролга мослашиш** | Декан фақат ўз факультетини, ўқитувчи фақат ўз фанларини кўради |

### 2.4 Техник архитектура

- **Контроллер:** `JournalController.php` (2,600+ сатр)
- **Асосий view файллар:**
  - `journal/index.blade.php` — фильтрлар саҳифаси
  - `journal/show.blade.php` — батафсил журнал кўриниши (138 КБ)
- **API эндпоинтлар:**
  - `GET /journal` — асосий саҳифа
  - `GET /journal/show/{groupId}/{subjectId}/{semesterCode}` — батафсил кўриниш
  - `POST /journal/save-mt-grade` — МТ баҳосини сақлаш
  - `POST /journal/save-retake-grade` — қайта топшириш баҳосини сақлаш
  - Фильтрлар учун 8+ AJAX эндпоинт

### 2.5 Маълумотлар оқими

```
student_grades жадвали
        │
        ├── training_type_code = 11+ → Амалий машғулот (ЖБ)
        ├── training_type_code = 99  → Мустақил таълим (МТ)
        │
        ▼
  Журнал кўриниши:
  ┌────────────────────────────────────────────────────┐
  │         │  01.02  │  03.02  │  05.02  │ Ўртача │ МТ │
  │ Талаба  │  ЖБ     │  ЖБ     │  ЖБ     │        │    │
  ├─────────┼─────────┼─────────┼─────────┼────────┼────┤
  │ Алиев   │   85    │   90    │   88    │  87.7  │ 80 │
  │ Каримов │   70    │   75    │   80    │  75.0  │ 65 │
  └────────────────────────────────────────────────────┘
```

## 3. МУСТАҚИЛ ТАЪЛИМ (МТ) БЎЛИМИ

### 3.1 Тавсиф
Мустақил таълим бўлими — талабаларга берилган **мустақил ишларни бошқариш**, уларнинг бажарилишини назорат қилиш ва баҳолаш учун мўлжалланган. Бу бўлим ўқитувчиларга топшириқлар бериш, талабаларга эса уларни бажариб топшириш имкониятини яратади.

### 3.2 Ишлаш принципи

```
┌─────────────────────────────────────────────────────────────────┐
│                    МУСТАҚИЛ ТАЪЛИМ ЖАРАЁНИ                       │
│                                                                  │
│  Ўқитувчи                     Талаба                            │
│  ┌─────────────┐              ┌─────────────┐                   │
│  │ Топшириқ    │              │ Топшириқни  │                   │
│  │ яратади     │ ──────────► │ кўради      │                   │
│  └─────────────┘              └──────┬──────┘                   │
│                                      │                           │
│                                      ▼                           │
│                               ┌─────────────┐                   │
│                               │ Файл юклаб  │                   │
│                               │ жавоб беради│                   │
│                               └──────┬──────┘                   │
│                                      │                           │
│                                      ▼                           │
│  ┌─────────────┐              ┌─────────────┐                   │
│  │ Баҳо қўяди  │ ◄─────────  │ Жавоб       │                   │
│  │ (0-100)     │              │ текширилади  │                   │
│  └──────┬──────┘              └─────────────┘                   │
│         │                                                        │
│         ▼                                                        │
│  ┌──────────────────────────────────────────┐                   │
│  │ Баҳо < 60?                               │                   │
│  │  ✓ Ҳа  → Талаба қайта юклаши мумкин     │                   │
│  │  ✗ Йўқ → Якунланган (≥60 бўлса блок)    │                   │
│  └──────────────────────────────────────────┘                   │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 Асосий имкониятлар

#### Ўқитувчи томонидан:

| Имконият | Тавсиф |
|----------|--------|
| **Топшириқ яратиш** | Фан, гуруҳ, семестр бўйича мустақил иш яратиш |
| **Файл бириктириш** | Топшириққа намуна файл (PDF, DOCX ва бошқа) бириктириш |
| **Муддат белгилаш** | Бошланиш ва тугаш муддатини белгилаш |
| **Баҳолаш** | Ҳар бир талабанинг жавобини кўриш ва баҳолаш (0-100) |
| **Қайта топшириш** | Баҳо < 60 бўлганда талабага қайта юклаш имконини бериш |
| **Тариxни кўриш** | Барча баҳо ва топшириш тарихини кўриш |

#### Талаба томонидан:

| Имконият | Тавсиф |
|----------|--------|
| **Топшириқларни кўриш** | Барча мустақил ишлар рўйхатини кўриш |
| **Файл юклаш** | Бажарилган ишни файл сифатида юклаш |
| **Статусни кўриш** | Баҳоланган/кутилмоқда/муддати ўтган |
| **Қайта юклаш** | Баҳо < 60 бўлса, қайта топшириш |

### 3.4 Муддат ва чекловлар тизими

```
Созламалар:
┌────────────────────────────────────────────┐
│ mt_deadline_time = 17:00                   │  ← Кунлик муддат вақти
│ mt_max_resubmissions = 3                   │  ← Максимал қайта юклаш
│ deadlines жадвали → курсга қараб муддат    │  ← Курсга мослашган
└────────────────────────────────────────────┘

Қоидалар:
• Талаба муддат кунидаги 17:00 гача файл юклаши керак
• Баҳо ≥ 60 бўлса — қайта юклаш блокланади
• Максимал 3 марта қайта юклаш мумкин (созланувчан)
• Ҳар бир қайта юклаш тарихга ёзилади
```

### 3.5 Маълумотлар базаси структураси

```
independents (Топшириқлар)
├── id, user_id
├── teacher_hemis_id, teacher_name
├── department_hemis_id, department_name
├── group_hemis_id, group_name
├── semester_hemis_id, semester_name, semester_code
├── subject_hemis_id, subject_name
├── start_date, deadline
├── file_path, file_original_name
├── grade_teacher, status (0=фаол, 1=нофаол)
└── created_at, updated_at

independent_submissions (Талабалар жавоблари)
├── id, independent_id, student_id, student_hemis_id
├── file_path, file_original_name
├── submitted_at (топшириш вақти)
├── submission_count (неча марта юклаган)
├── viewed_at (ўқитувчи кўрган вақт)
└── UNIQUE: [independent_id, student_id]

independent_grade_history (Баҳо тарихи)
├── id, independent_id, student_id
├── grade (0-100)
├── submission_number (қайта юклаш рақами)
├── graded_by, graded_at
└── created_at, updated_at
```

### 3.6 Техник архитектура

- **Контроллерлар:**
  - `IndependentController.php` — админ/ўқитувчи (400+ сатр)
  - `StudentController.php` — талаба портали
- **View файллар:**
  - `independent/index.blade.php` — рўйхат (24 КБ)
  - `independent/create.blade.php` — яратиш формаси (31 КБ)
  - `independent/grade.blade.php` — баҳолаш (12 КБ)
  - `independent/grade_form.blade.php` — баҳо формаси (14 КБ)
  - `student/independents.blade.php` — талаба кўриниши (15 КБ)
- **API эндпоинтлар:**
  - `GET /independent` — рўйхат
  - `POST /independent/store` — яратиш
  - `GET /independent/grade/{id}` — баҳолаш
  - `POST /independent/grade` — баҳо сақлаш
  - `POST /student/independents/{id}/submit` — талаба топшириши

### 3.7 Фаолият журнали (Activity Log)
Мустақил таълим бўлимидаги барча амаллар тизимда қайд этилади:
- Топшириқ яратиш/таҳрирлаш/ўчириш
- Талабанинг файл юклаши
- Баҳо қўйиш ва ўзгартириш
- Қайта топшириш амаллари

Бу **шаффофлик ва назорат** учун муҳим — ким, қачон, нима қилгани аниқ кўринади.

## 4. ҲИСОБОТЛАР БЎЛИМИ

### 4.1 Тавсиф
Ҳисоботлар бўлими — ўқув жараёни ҳақида **таҳлилий ва йиғма маълумотлар** шакллантириш учун мўлжалланган. Бу бўлим маъмурият ва ўқитувчиларга талабалар ўзлаштириши, давомати, ўқув юклама ва бошқа кўрсаткичларни кузатиш имконини беради.

### 4.2 Ҳисобот турлари

Тизимда **7 та асосий ҳисобот тури** мавжуд:

```
ҲИСОБОТЛАР
├── 1. ЖН ўзлаштириш ҳисоботи
├── 2. Дарс тақсимоти ҳисоботи
├── 3. Дарс жадвали ҳисоботи
├── 4. Давомат (сабабсиз) ҳисоботи
├── 5. Юклама ва жуфтлик ҳисоботи
├── 6. Қарздорлар ҳисоботи
└── 7. Сабабли текшируви ҳисоботи
```

---

#### 4.2.1. ЖН ўзлаштириш ҳисоботи (JN Report)

**Мақсад:** Талабаларнинг амалий машғулотлар (ЖБ) бўйича ўзлаштириш даражасини аниқлаш.

**Ишлаш принципи:**
```
Кунлик баҳолар → Кунлик ўртача → Умумий ўртача → Ҳисобот
```

| Параметр | Тавсиф |
|----------|--------|
| Фильтрлар | Таълим тури, факультет, йўналиш, курс, семестр, гуруҳ, фан |
| Сана оралиғи | Бошланиш ва тугаш санасини белгилаш мумкин |
| Натижа | Ҳар бир талаба бўйича ўртача баҳо ва ўзлаштириш фоизи |
| Саҳифалаш | Катта маълумотлар учун пагинация |

---

#### 4.2.2. Дарс тақсимоти ҳисоботи (Lesson Assignment)

**Мақсад:** Қайси фан қайси гуруҳга, қайси ўқитувчи томонидан ўтилаётганини кўрсатиш.

| Параметр | Тавсиф |
|----------|--------|
| Фильтрлар | Факультет, кафедра, йўналиш, курс, семестр |
| Натижа | Фан-ўқитувчи-гуруҳ боғланиши |
| Фойдаси | Дарс тақсимотини назорат қилиш, ортиқча/камчилик аниқлаш |

---

#### 4.2.3. Дарс жадвали ҳисоботи (Schedule Report)

**Мақсад:** Ўқув жадвали бўйича батафсил маълумот бериш.

| Параметр | Тавсиф |
|----------|--------|
| Маълумот | Дарс саналари, вақтлари, фанлар, ўқитувчилар, гуруҳлар |
| Фильтрлар | Кенг фильтрлаш имконияти |
| Натижа | Жадвал ижроси мониторинги |

---

#### 4.2.4. Давомат (сабабсиз) ҳисоботи (Absence Report)

**Мақсад:** Талабаларнинг **сабабсиз қолдирган дарслари** ҳақида маълумот.

| Параметр | Тавсиф |
|----------|--------|
| Фильтрлар | Талаба, фан, сана оралиғи |
| Маълумот | Қолдириш кодлари ва сони |
| Фойдаси | Интизомий назорат, огоҳлантириш тизими |

---

#### 4.2.5. Юклама ва жуфтлик ҳисоботи (Load vs Pair)

**Мақсад:** Ўқитувчиларнинг **ўқув юкламаси таҳлили** — режалаштирилган юклама ва амалда ўтилган дарслар таққослаш.

| Параметр | Тавсиф |
|----------|--------|
| Фильтрлар | Факультет, ўқитувчи |
| Маълумот | Белгиланган юклама vs реал дарс жуфтликлари |
| Фойдаси | Ўқитувчи иш юкини мувозанатлаш, ресурсларни бошқариш |

---

#### 4.2.6. Қарздорлар ҳисоботи (Debtors)

**Мақсад:** **Баҳоси тўлиқ бўлмаган** (қарздор) талабалар рўйхатини шакллантириш.

| Параметр | Тавсиф |
|----------|--------|
| Маълумот | Қайси фанлардан қарзи бор, ҳолати |
| Фильтрлар | Факультет, гуруҳ, фан |
| Фойдаси | Семестр якунида қарздорларни аниқлаш ва назорат қилиш |

---

#### 4.2.7. Сабабли текшируви ҳисоботи (Sababli Check)

**Мақсад:** **Сабабли деб белгиланган** қолдиришларни текшириш — ҳужжат мавжудлигини назорат қилиш.

| Параметр | Тавсиф |
|----------|--------|
| Маълумот | Сабабли белгиланган қолдиришлар |
| Текшириш | Ҳужжат борлигини тасдиқлаш |
| Фойдаси | Сохта сабабларнинг олдини олиш |

---

```
                                 │ attendances    │
                                 │ schedules      │
                                 │ ва бошқалар    │
                                 └──────────────┘
```

**Техник хусусиятлар:**

- **Контроллер:** `ReportController.php` (2,600+ сатр)
- **View файллар:** Ҳар бир ҳисобот учун алоҳида blade шаблон (27-40 КБ)
- **Маълумот юклаш:** AJAX — саҳифа юкланганда фақат UI, маълумот талаб бўйича юкланади
- **Фильтрлаш:** Каскадли фильтрлар (факультет → йўналиш → курс → семестр → гуруҳ → фан)
- **Рол чеклови:** Декан фақат ўз факультети маълумотларини кўради
- **Вақт минтақаси:** Тошкент вақти (Asia/Tashkent)
- **Экспорт:** Excel форматида юклаб олиш имконияти

- **Хавфсизлик** — роллар, ҳуқуқлар ва фаолият журнали орқали шаффофлик
